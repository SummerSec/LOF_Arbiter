# LOF Arbiter - 套利精灵 Skill

> LOF 基金溢价套利机会监测与交互式咨询
> 
> **版本**: 2026.6.3
> 
> **特性**: 独立运行，不依赖 DataHub

---

## 功能说明

本 Skill 帮助用户查询和分析 LOF 基金的折溢价套利机会。

### 核心能力

1. **限购高溢价查询** - 筛选限购且高溢价的优质套利机会
2. **高溢价排行** - 查询卖出赎回套利机会 TOP N
3. **高折价排行** - 查询买入套利机会 TOP N
4. **单基金诊断** - 输入基金代码/名称，返回详细信息和分析
5. **套利收益测算** - 计算扣除费用后的理论套利收益
6. **CSV导出** - 导出 LOF 基金行情列表

### 数据来源（独立）

- **自有 SQLite 数据库**：`data/lof_arbiter.db`
- **ETL 数据源**：
  - 新浪财经（LOF 实时行情）
  - 东方财富（基金净值、申购状态）

## 核心套利逻辑

### 1. 限购溢价优先原则

> **限购限大额 + 高溢价 = 优质套利机会**

原因：
- 限购产品溢价更稳定，不易被套利者瞬间抹平
- 无限购的产品流动性太好，溢价会被快速消除
- 高溢价代表更大的套利空间

### 2. 流动性门槛

| 条件 | 说明 |
|------|------|
| 成交额 ≥ 100万 | 最低流动性门槛 |
| 排除暂停申购 | 无法进行套利操作 |

### 3. 溢价率计算（T-2 锚定 + 实时估算）

#### 核心概念

场内 QDII（LOF/ETF）有两个价格：

| 价格 | 含义 |
|------|------|
| 场内市价 | 股票账户实时买卖价，由供需决定 |
| 基金净值 | 真实资产价值，由海外持仓决定 |

- **溢价**：市价 > 预估净值 → 买贵了，谨慎入场
- **折价**：市价 < 预估净值 → 存在套利空间

#### QDII 致命规则：净值天然滞后

国内 A 股 T 日能看到的官方净值，反映的是 **T-2 日**（或更早）海外持仓，没有任何软件能给出 T 日真实净值，盘中溢价全部是估算值。

```
T-2 日 → 基金公司公布精准官方净值 N(T-2)（唯一真实锚点）
T-1 日 → 海外市场涨跌，需手动估算
T 日   → A 股交易，盘中涨跌只影响下一个交易日净值
```

#### 万能计算公式

```
T日基准预估净值 = N(T-2) × (1 + R(T-1)/100 × 仓位) × (1 + 汇率(T-1)/100)
当日溢价率     = (场内现价 - 基准预估净值) / 基准预估净值 × 100%
T+1预估净值    = 基准预估净值 × (1 + R(T)/100 × 仓位)
次日预估溢价率 = (现价 - T+1预估净值) / T+1预估净值 × 100%
```

#### 港股 vs 美股：精度天差地别

| 类型 | 当日溢价 | 次日预估溢价 | 标记 |
|------|----------|--------------|------|
| 港股 QDII（`TRACKING_HK`） | ✅ 较精准 | ✅ 较精准 | 高置信 |
| 美股 QDII（`TRACKING_US`） | ✅ 较精准（依托前一晚美股） | ❌ 不可精准 | 仅当日可信 |
| 国内 LOF（`TRACKING_DOM`） | ✅ 当日估算 | 参考 | 高置信 |
| 混合/商品（`TRACKING`） | ⚠ 估算 | ⚠ 仅供参考 | 低置信 |

**美股 QDII 关键真相**：A 股 9:30–15:00 交易时美股休市，当日溢价用前一晚落地行情可算准；次日溢价 100% 是期货/情绪预判，今晚美股大幅波动后全部作废。**请勿依据 APP「次日预估溢价」交易。**

**港股 QDII 优势**：交易时间与 A 股高度重合，T 日指数涨跌收盘即落地，当日和次日溢价均可较精准计算。

#### 锚定净值 N(T-2) 取值

| `nav_lag_days` | 锚点来源 | 说明 |
|----------------|----------|------|
| 0 | 最新官方净值 `nav` | 多数 QDII 最新净值已是 T-2 数据 |
| ≥1 | 上一档 `prev_nav` | 基金披露额外滞后时使用 |

#### 实操案例（164906 中概互联，2025-06-03）

```
N(T-2) = 0.9799    R(T-1) = +3.86%    R(T) = -2.5%    收盘价 P = 0.988

T日基准预估净值 = 0.9799 × 1.0386 ≈ 1.0177
当日溢价率     = (0.988 - 1.0177) / 1.0177 ≈ -2.92%（折价）
T+1预估净值    = 1.0177 × 0.975 ≈ 0.9923
次日预估溢价   = (0.988 - 0.9923) / 0.9923 ≈ -0.43%
```

> 汇率等因素会导致与实盘略有偏差。嫌麻烦可直接看「当日溢价」，毕竟场内明日涨跌无法预知。

#### 五级估算策略

| 级别 | 条件 | 标记 | 说明 |
|------|------|------|------|
| `TRACKING_HK` | 港股同步 QDII | 高置信 | 展示当日 + 次日预估溢价 |
| `TRACKING_US` | 美股 QDII | 当日可信 | 不展示次日预估（或标红警告） |
| `TRACKING_DOM` | 国内 LOF | 高置信 | 标准实时估算 |
| `LEGACY` | 无配置或无基准 | 不可用 | 原 (price-nav)/nav |
| `NONE` | 无净值数据 | — | premium=NULL |

#### 国内 LOF 与国内 QDII 公式

```
// 国内 LOF
T日基准预估净值 = 昨日净值 × (1 + 跟踪标的涨跌幅 × 仓位比例)

// QDII LOF
T日基准预估净值 = N(T-2) × (1 + R(T-1) × 仓位比例) × (1 + USD/CNY 涨跌幅)

溢价率 = (场内价格 - T日基准预估净值) / T日基准预估净值 × 100%
```

#### 投资避坑准则

1. **港股 QDII**：溢价数据较可信，出现机会可把握
2. **美股 QDII**：只看当日收盘溢价；彻底忽略「次日预估溢价」
3. **混合基金**（同时含美股+港股）：计算复杂，T-1 溢价较大时再操作
4. **必须展示净值日期**，用于判断数据时效性

### ODS 层数据来源

| 数据源 | 提供字段 |
|--------|----------|
| 新浪财经 `fund_etf_category_sina` | LOF 基金实时行情：代码、名称、现价、涨跌幅、成交额 |
| 东方财富 `fund_open_fund_daily_em` | 基金净值：第3列（最新）、第5列（上一交易日） |
| 东方财富 `fund_purchase_em` | 申购状态、手续费、购买起点 |
| akshare `stock_zh_index_spot_em` | A 股指数实时涨跌幅（跟踪标的） |

| akshare `forex_spot_em` | 实时汇率（QDII 需要） |

### 基金跟踪配置

`data/fund_tracking_config.json` 维护 LOF 基金的跟踪标的映射关系，
包含：基金代码、基金类型（QDII/DOMESTIC）、跟踪标的代码与名称、仓位比例、汇率对等。

### 数据库表结构

```sql
lof_daily (
    fund_code,           -- 基金代码
    fund_code_full,      -- 标准代码 XXXXXX.SZ/SH
    fund_name,           -- 基金名称

    nav_date,            -- 净值日期

    benchmark_change_t0, -- R(T) 跟踪标的涨跌幅(%)

```

## 输出格式规范

所有输出**必须包含**：

1. **净值日期** - 溢价率计算的基础，非常重要
2. **溢价率颜色标注**：
   - 🔥 溢价 > 1%：适合卖出赎回
   - 💎 折价 < -1%：适合买入
   - 📊 正常 ±1% 以内：观望
3. **QDII 精度标注**（必须展示）：
   - `[港股同步·高置信]`：当日 + 次日溢价均可参考
   - `[美股QDII·当日可信]`：仅当日溢价，不展示次日预估
   - `[Legacy]`：无跟踪配置，基于官方净值
4. **状态标注**：
   - [限购]：限购产品，溢价更稳定
   - [暂停]：暂停申购，无法套利

## 注意事项

1. **T+2 交割**：LOF 套利从买入到资金到账需 2 个交易日
2. **净值滞后**：QDII 官方净值天然滞后 2 天，溢价率为 T-2 锚定估算值
3. **美股 QDII**：仅「当日溢价」可信，次日预估不可作为交易依据
4. **港股 QDII**：当日/次日溢价均可较精准计算
5. **流动性风险**：高溢价不等于能成交，需关注成交额
6. **限购优先**：限购产品溢价更稳定，优先关注
7. **净值日期**：必须显示，用于判断数据时效性
